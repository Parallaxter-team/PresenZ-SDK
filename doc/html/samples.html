Render Samples

A PzRenderSample represents one shaded sample in the PresenZ Render phase.
It contains all the information needed for correct pixel reconstruction, depth ordering, special material handling (e.g., glass), motion blur, and custom AOVs.

You build and fill PzRenderSample instances in your shading/filter stage, then either:

  • Push them directly to PresenZ in bucket mode via PzProcessRenderSample(), or
  • Pass them to PzFilterRenderSample() in no-bucket mode to produce a compact, driver-ready packet.

9.1 Standard attributes

Function Type Description
PzSetSamplePosition(sample, NozVector) World-space position 3D hit point of the sample in world coordinates. Used for depth sorting and reconstruction.
PzSetSampleNormal(sample, NozVector) World-space normal Surface shading normal at the hit point.
PzSetSampleZ(sample, float) Depth value Distance from camera/ZOV to the hit point. Special values: NOZ_INFINITE for background/matte samples.
PzSetSamplePosXY(sample, float, float) Relative pixel position Sub-pixel coordinates inside the pixel (0–1 range).
PzSetSampleDepth(sample, int) Ray depth Recursion depth in the ray tree; 0 for rays that didn't went through any (semi-)transparent surface/volume.

9.2 Color and shading

Function Type Description
PzSetSampleColor(sample, NozRGBA) RGBA color Final shaded color of the sample.
PzAddGlassSampleAov(sample, int) Glass flag Marks the sample as glass (1) or non-glass (0). Used for correct compositing of transparent surfaces.
PzAddVelocitySampleAov(sample, NozVector) Motion vector Optional per-sample motion vector for motion blur.

9.3 Special material handling

Glass samples:
If the surface is flagged as glass (ie. prz_transp tag in Arnold), you may:

  • Compute transmission from a "transmission" AOV.
  • Override the sample’s RGBA and alpha to represent attenuation through glass.
  • Chain transmission across depths: multiply by previous lastTransmission.

Dome/max clip plane samples:
If the hit distance ≥ max_distance (near far clip), you may recompute position/normal using a virtual sphere hit test to avoid incorrect depth.

Matte objects:
If the hit distance is 0.0f, mark the sample as background with NOZ_INFINITE depth.

9.4 Cryptomatte IDs

When cryptomatte output is enabled:

Function Type Description
PzSetSampleCrypto(sample, cryptoType::object, float) Object ID Cryptomatte object hash.
PzSetSampleCrypto(sample, cryptoType::material, float) Material ID Cryptomatte material hash.
PzSetSampleCrypto(sample, cryptoType::asset, float) Asset ID Cryptomatte asset hash.

Only set the cryptomatte types enabled in your render (PzGetActiveCryptomatte()).

9.5 Arbitrary Output Variables (AOVs)

You can attach any number of extra AOV values to the sample:

Function Type Description
PzAddSampleAov(sample, void* valuePtr, const std::string& name) Any supported type Adds a named AOV to the sample. Supported types: INT, BOOL, FLOAT, RGB, RGBA, VEC, VEC2.

In Arnold, these values are fetched from the AOV iterator:

if (type == PresenZ::Phase::AOVType::FLOAT) {
auto value = AiAOVSampleIteratorGetAOVFlt(iterator, AOVname);
PzAddSampleAov(sample, (void*)&value, name);
}

‍Tip: Avoid adding PresenZ internal AOVs (e.g., N, P, motionVector, cryptomattes) as regular sample AOVs.

9.6 Construction pattern

A typical PzRenderSample setup:

PzRenderSample sample;
// Geometry
PzSetSamplePosition(sample, worldPos);
PzSetSampleNormal(sample, worldNormal);
PzSetSampleZ(sample, zValue);
PzSetSamplePosXY(sample, offset.x + 0.5f, offset.y + 0.5f);
PzSetSampleDepth(sample, rayDepth);
// Color
PzSetSampleColor(sample, NozRGBAFromAiRGBA(RGBA));
// Optional: glass handling, cryptomatte, velocity
PzAddGlassSampleAov(sample, isGlass ? 1 : 0);
if (hasVelocity) PzAddVelocitySampleAov(sample, velocityVec);
if (hasCryptoObject) PzSetSampleCrypto(sample, cryptoType::object, objectHash);
// Custom AOVs
PzAddSampleAov(sample, (void*)&myValue, "my_custom_aov");

Once filled, either:

  • Bucket mode: push immediately with PzProcessRenderSample().
  • No-bucket mode: append to a vector for the pixel, then call PzFilterRenderSample() and later PzFlushSamplesToFile().
