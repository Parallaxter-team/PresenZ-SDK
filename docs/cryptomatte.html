Cryptomatte Support

PresenZ integrates Cryptomatte to store object, material, and asset IDs per sample in the .przRender output.
These IDs can be used in compositing to generate accurate mattes for scene elements.

10.1 Overview

Cryptomatte in PresenZ:

  • Is optional and configured at render time via PzSetCryptomatte().
  • Supports three ID types:
    • Object (cryptoType::object)
    • Material (cryptoType::material)
    • Asset (cryptoType::asset)
  • Produces:
    • Per-sample ID hashes and weights (stored in the render samples)
    • Manifests mapping hash â†’ original name

The integration consists of:

  1. Configuration
  2. ID extraction in the shader (PzProcessNode()).
  3. Sample tagging in the filter (PzSetSampleCrypto()).
  4. Manifest compilation at render end (PzAddObjectToManifest(), PzAddMaterialToManifest()).

10.2 Enabling Cryptomatte

At PresenZ initialization, call:

PzSetCryptomatte(
do_crypto_object,
do_crypto_material,
do_crypto_asset,
depth_object,
depth_material,
depth_asset
);

This call tells PresenZ which ID types to output and how many layers deep to store (one depth = 2 layers). Because of the volumetric nature of PresenZ, more than 2 depths is rarely necessary.

10.2 Extracting IDs in the shader

In the Render pass, if Cryptomatte is enabled, uses the API to query and hash the current object/material/asset names:

PzProcessNode(&getCryptoInfo, threadId, nsp_hash, obj_hash, mat_hash);

getCryptoInfo is a callback that will be used to retrieve the geometry/material name, depending of the PzSetCryptomatte() enabled options. The threadId is used for internal name caching. It should match the current thread used by the sample.

10.4 Attaching IDs to render samples

Write the values returned by PzProcessNode to the render sample:

bool do_object, do_material, do_asset = false;
PzGetActiveCryptomatte(do_object, do_material, do_asset);
if (do_object)
PzSetSampleCrypto(sample, cryptoType::object, valueObj);
if (do_material)
PzSetSampleCrypto(sample, cryptoType::material, valueMaterial));
if (do_asset)
PzSetSampleCrypto(sample, cryptoType::asset, valueAsset));
}

10.5 Writing manifests

If any Cryptomatte type was active, build the manifests. You can do it at any convenient time, eg. scene loading/construction or at the end of the render.

Calls PzAddObjectToManifest(objectName) for each object. Calls PzAddMaterialToManifest(materialName) for each assigned shader (if material cryptomatte is enabled).

PresenZ::Cryptomatte::PzAddObjectToManifest("Object Name");
PresenZ::Cryptomatte::PzAddMaterialToManifest("Material Name");
PresenZ::Cryptomatte::v5_0::PzAddMaterialToManifest
void PzAddMaterialToManifest(const char *name)
PresenZ::Cryptomatte::v5_0::PzAddObjectToManifest
void PzAddObjectToManifest(const char *name)

These manifests are written into the output for compositing tools to decode the hashes back to names.
